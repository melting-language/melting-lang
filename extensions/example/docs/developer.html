Melt extension developer guide

How to write loadable extensions (shared libraries) for the Melt interpreter.

1. Extension contract

Your shared library must export exactly one symbol:

extern "C" void melt_register(Interpreter* interp);

When Melt loads the extension, it calls melt_register(interp). Use it to register native built-in functions with:

interp->registerBuiltin("functionName", yourNativeFn);

2. Built-in function signature

Each built-in has type Interpreter::NativeFn:

Value yourBuiltin(Interpreter* interp, std::vector<Value> args);

Value is a std::variant holding: std::monostate, double, std::string, bool, and object/array/class types. Return a Value (e.g. Value(3.14), Value(std::string("hi"))).

3. Include path

Compile with the same include path as the main binary so #include "interpreter.hpp" resolves (e.g. -I src -I src/core from the repo root).

4. Building

  • From repo root: make modules builds the example extension into bin/modules/ (example.dylib on macOS, example.so on Linux, example.dll on Windows).
  • macOS: use -fPIC -shared -undefined dynamic_lookup so the extension resolves Interpreter::registerBuiltin from the main binary at load time.
  • Linux: the main binary is built with -rdynamic so loaded .so files can resolve interpreter symbols.

5. Configuration (melt.ini)

In bin/melt.ini (or project melt.config):

  • extension_enabled = 1|0|true|false|on|off — enable or disable loading extensions (default: on).
  • extension_dir = directory relative to the melt binary (default: modules).
  • extension = comma-separated list of extension names to load (e.g. example, math).

Extensions are loaded from <binDir>/<extension_dir>/<name>.so|.dylib|.dll.

6. Example source

See extensions/example/example.cpp for a minimal extension that registers exampleVersion, exampleAdd, and exampleSqrt.

7. Adding your own extension

  1. Create a folder under extensions/ (e.g. extensions/mymod/) with a single .cpp implementing melt_register and registerBuiltin calls.
  2. Add a Makefile target (or reuse the pattern from example) to build a shared library into bin/modules/mymod.so (or .dylib/.dll).
  3. Set extension = mymod (or append , mymod) in melt.ini.
